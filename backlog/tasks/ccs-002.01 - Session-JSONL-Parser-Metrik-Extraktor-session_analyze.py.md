## Final Summary

<!-- SECTION:FINAL_SUMMARY:BEGIN -->
Implemented scripts/session_analyze.py and scripts/test_session_analyze.py.

The script derives the project session dir from CWD via path encoding (/ → -), reads all *.jsonl in that dir, and extracts:
- failed_commands: tool_results with is_error=true, correlated back to their tool_use for name/command/session_id
- tool_frequencies: Counter over all tool_use entries across sessions
- token_stats: input/output/cache_read/cache_creation sums plus per-session breakdown
- repeated_sequences: sliding-window detection of repeating tool-call subsequences (min_len=2, min_count=2)

Tests: 11 pytest tests using fixture JSONLs — all RED confirmed before implementation, all GREEN after. just test remains green (10 sprint_bridge tests unaffected).
